1. 배경: 왜 수정이 필요했는가

LSTM과 데이터 특성의 충돌

LSTM은 시간 연속성(temporal context)을 학습하는 모델이다. 기존 ResNet 실험에서 사용한 ratio_ver 데이터는 클래스 균형을 맞추기 위해 에포크를 랜덤으로 제거(undersampling)했는데, 이렇게 하면 시간 흐름이 끊겨서 LSTM의 시퀀스 생성이 불가능해진다.

따라서 LSTM은 full_ver 데이터를 그대로 사용해야 하며, 클래스 불균형은 데이터가 아닌 학습 과정에서 보정해야 한다.

full_ver train 클래스 분포

클래스개수비율문제점
nrem16,28389.0%압도적 다수
wake1,1986.6%소수
rem8094.4%극소수

기존 코드는 nn.CrossEntropyLoss()를 가중치 없이 사용해서, 모델이 전부 nrem으로 예측해도 89% 정확도가 나오는 상태였다.

2. 버그 수정

Bug #1: CNN 가중치 로드 시 shape mismatch 에러

pytorch/models_lstm.py : load_cnn_weights()

문제 상황:

  • 2클래스 CNN 체크포인트의 fc_audioset 출력: [2048, 2]
  • LSTM 내부 CNN의 fc_audioset 기대: [2048, 3] (3클래스로 생성 시)
  • load_state_dict()에서 shape 불일치로 즉시 에러
  • 실제로 LSTM은 CNN의 embedding(2048차원)만 사용하고 fc_audioset은 사용하지 않음

수정 전:

checkpoint = torch.load(checkpoint_path, map_location=device, weights_only=False)
self.cnn.load_state_dict(checkpoint['model'])  # shape 불일치 시 에러

수정 후:

pretrained_dict = checkpoint['model']
model_dict = self.cnn.state_dict()

loaded = {}
skipped = []
for k, v in pretrained_dict.items():
    if k in model_dict and v.shape == model_dict[k].shape:
        loaded[k] = v
    else:
        skipped.append(k)

model_dict.update(loaded)
self.cnn.load_state_dict(model_dict)

shape이 맞는 레이어만 로드하고, 불일치 레이어(fc_audioset)는 스킵. 스킵된 레이어 목록을 출력하여 디버깅 용이.

Bug #2: Focal Loss 호출 인터페이스 불일치

pytorch/main_lstm.py : train()

문제 상황:

  • main_lstm.py의 호출 방식: loss_func(logits, target)
  • losses.py의 FocalLoss.forward() 기대: loss_func(output_dict, target_dict)
  • --loss_type=focal 선택 시 런타임 에러 발생

수정: FocalLoss를 래핑하여 호출 인터페이스 통일

from losses import FocalLoss
_focal = FocalLoss(weight=weight_tensor, gamma=2.0)
loss_func = lambda logits, target: _focal(
    {'clipwise_output': logits}, {'target': target})

3. 기능 추가

Feature #1: Class-weighted CrossEntropyLoss (기본값)

pytorch/main_lstm.py : --loss_type=weighted_ce

역빈도(inverse frequency) 가중치를 자동 계산하여 CrossEntropyLoss에 적용.

기본값으로 설정되어 별도 옵션 없이도 자동 적용됨.

클래스개수가중치효과
wake1,198~5.09wake 1개 = nrem ~14개 가치
rem809~7.54rem 1개 = nrem ~20개 가치
nrem16,283~0.37다수 클래스 페널티 감소

계산 공식: weight_i = total / (n_classes * count_i)

Feature #2: Focal Loss 옵션

pytorch/main_lstm.py : --loss_type=focal

클래스 가중치 + Focal Loss(gamma=2.0) 조합. 이미 잘 맞추는 쉬운 샘플(nrem)의 loss를 줄이고, 어려운 샘플(rem, wake 전환 구간)에 집중하게 함.

기존 losses.py의 FocalLoss 클래스를 재활용.

Feature #3: WeightedRandomSampler

pytorch/main_lstm.py : --oversample

시퀀스의 중간 라벨 기준으로 가중 샘플링. rem/wake 시퀀스를 더 자주 뽑되, 각 시퀀스 내부의 시간 연속성은 그대로 유지.

data_generator_lstm.py에 이미 구현되어 있던 get_sample_weights()를 활용.

핵심: ratio_ver처럼 에포크를 삭제하는 것이 아니라, 샘플링 확률만 조정하므로 시간 흐름이 보존됨.

4. 검증 Loss 분리

pytorch/main_lstm.py : train() 내 검증 단계

학습에는 가중치가 적용된 loss(weighted_ce/focal)를 사용하지만, 검증(val)에는 기본 CrossEntropyLoss를 사용하여 공정한 비교가 가능하도록 변경.

# 검증 (공정한 비교를 위해 기본 CE 사용)
val_loss_func = nn.CrossEntropyLoss()
val_acc, val_loss_avg = evaluate(model, val_loader, val_loss_func, device)

이렇게 해야 loss_type을 바꿔가며 실험할 때 val loss 수치를 직접 비교할 수 있음.

6. Loss 옵션 비교 가이드

옵션클래스 가중치어려운 샘플 집중권장 상황
--loss_type=clip_ce X X 이미 밸런싱된 데이터 (ratio_ver)
--loss_type=weighted_ce O (역빈도) X 기본값. full_ver 불균형 데이터에 적합
--loss_type=focal O (역빈도) O (gamma=2.0) 불균형 + 클래스 간 난이도 차이가 클 때

--oversample은 위 옵션과 독립적으로 조합 가능. 권장 조합:

  • 기본: --loss_type=weighted_ce --oversample
  • 공격적: --loss_type=focal --oversample
